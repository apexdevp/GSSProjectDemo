A few terms worth knowing

  • commit — a saved snapshot of your files with a message.
  • branch — a separate line of work you can develop without touching main.
  • merge — combining one branch's changes into another.
  • diff — the line-by-line difference between two versions.
  • remote — a copy of the repository hosted elsewhere, e.g. on GitHub.

Push it somewhere

When you're ready to back up your work or share it, connect a remote and push. Create an empty repo on a host first, then link and upload your commits with git push -u origin main.
